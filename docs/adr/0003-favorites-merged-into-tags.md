# ADR-0003: お気に入り機能 — 専用カラム廃止、タグシステムに統合

## Context

当初 `is_favorite` カラムを images テーブルに持つ設計だった。

## Decision

お気に入り専用のカラム・API・コンポーネントを全廃し、タグシステムに統合する。予約タグも設けず、完全にユーザー任せとする。

## Alternatives

| 選択肢                                                       | 利点                            | 欠点                                                           | 不採用理由                                                                                               |
| ------------------------------------------------------------ | ------------------------------- | -------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------- |
| is_favorite カラム + 専用API + FavoriteToggle コンポーネント | シンプルな UI、ワンクリック操作 | スキーマ・API・UI コンポーネントが増える、概念としてタグと重複 | お気に入りは「特別なタグ」でしかなく、タグに統合すれば「要確認」「ボツ」等の運用タグも同じ仕組みで回せる |

## Consequences

**Positive:**

- 概念の重複が解消され、コードベースがシンプルになる
- タグに統合することで、お気に入り以外の運用タグも同じ仕組みで回せる
- 予約タグを設けないことで、ユーザーの自由度が最大化される

**Negative:**

- お気に入りのワンクリック操作がなくなり、タグ付け操作が必要になる
